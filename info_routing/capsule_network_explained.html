1 Why CNNs aren't enough

A CNN is great at saying "a cat is here." But it throws away where and in what pose — and that's the problem capsules fix.

Translation invariance vs. equivariance

PropertyWhat it meansWho has it
InvarianceShift the object → output is unchanged ("still a cat"). The position information is lost.CNN + max-pooling
EquivarianceShift/rotate the object → the representation changes in step, so pose is preserved.Capsules

The classic problem

Max-pooling makes a CNN translation-invariant: a face with the mouth above the eyes still fires "face!", because pooling only asks "is the feature present somewhere?" — not "is it in the right place relative to the others?" CNNs also need lots of data to learn every viewpoint, because they don't model pose explicitly.

Capsules keep the part–whole spatial relationship: a "mouth" capsule and an "eye" capsule must agree on the pose of the face before the network says "face." That agreement check is the whole point of routing (Section 7).

2 What is a capsule?

A capsule is a small group of neurons whose output is a vector \(u=(m,\theta)\) — not a single number.

A normal neuron → a scalar

Output = one number from \(f(w\cdot x+b)\). It can only say "how much" of a feature is present. A scalar has no direction, so it cannot describe orientation, thickness, or size.

A capsule → a vector

Length \(\;\|u\|\in[0,1]\) = probability the entity is present.
Direction = the pose (width, height, angle, colour, …).
Same conv math — we just group outputs into a vector.

Try it: the capsule as a compass needle

Drag the sliders. Length = "is it present?" · angle = "what pose?". Rotating the input only rotates the needle — the length (probability) stays — that's equivariance.


Read-out: \(u=(m,\theta)\) →

Existence property: the capsule answers two questions at once — (1) is it present? from the length, and (2) what does it look like? from the direction. A scalar neuron can only answer the first.

3 Squashing — the capsule's activation

ReLU/sigmoid squashes a number. Squash squashes a vector's length into \([0,1)\) while keeping its direction.

$$v=\underbrace{\frac{\|s\|^{2}}{1+\|s\|^{2}}}_{\text{shrinks the length} \to [0,1)}\cdot\underbrace{\frac{s}{\|s\|}}_{\text{keeps the direction}}$$
CaseWhat happens
\(\|s\|\) large (≫1)\(\dfrac{\|s\|^2}{1+\|s\|^2}\to 1\): output length → just under 1 ("definitely present").
\(\|s\|\) small (→0)factor → 0: output length → 0 ("not present"). If \(s=0\), \(v=0\).

Try it: input length → output length

Slide the raw length \(\|s\|\). Notice the output never exceeds 1, and direction is untouched.

Output length \(\|v\| = \dfrac{\|s\|^2}{1+\|s\|^2}\) = 0.800

5 Primary capsules — making 8-D vectors

Key insight: there's no magic layer. A normal convolution outputs 256 channels; we just group them into 32 sets of 8, and each set of 8 numbers is a capsule.

Watch: 256 channels → 32 capsules of 8-D

Press play. The 256 conv outputs (top) get sliced into groups of 8 (bottom). Each coloured group at one pixel = one 8-D capsule.

The count

After stride-2 conv the map is \(6\times6\). We chose 32 capsule types, each 8-D (\(32\times8=256\) channels). So: $$\underbrace{32}_{\text{types}}\times\underbrace{6\times6}_{\text{locations}} = \mathbf{1152}\ \text{primary capsules, each 8-D}$$ Then every 8-D vector is passed through squash.
Say it simply: "A convolution gives 256 feature maps; we read every 8 of them at a pixel as one vector. That's the capsule — 8 filter responses stacked into a direction, then squashed."

6 Predictions — every part votes for every whole

Each primary capsule \(i\) predicts each digit capsule \(j\) through a learned transform \(W_{ij}\) (shape \(16\times8\)), turning its 8-D vector into a 16-D vote.

$$\hat u_{j|i}=W_{ij}\,u_i\qquad\text{(8-D part}\;\to\;\text{16-D prediction of the whole)}$$

What the vote means

\(\hat u_{j|i}\) says: "based on the part I detected (e.g. a curve), if digit \(j\) is here, its full pose should be this 16-D vector." When many parts predict the same pose for a digit, they agree — and that's the signal routing looks for.

Total predictions: \(1152 \times 10 = 11{,}520\) vectors, each 16-D.

7 Dynamic Routing-by-Agreement live demo

This is the heart of the model. Parts vote; we strengthen the connections whose votes agree with the emerging consensus — and repeat 3 times.

The algorithm (Sabour et al., 2017)

#StepMeaning
0\(b_{ij}\leftarrow 0\)start with no preference
1\(c_{ij}=\mathrm{softmax}_j(b_{ij})\)coupling — each part splits its trust over classes, \(\sum_j c_{ij}=1\)
2\(s_j=\sum_i c_{ij}\,\hat u_{j|i}\)weighted sum of votes for class \(j\)
3\(v_j=\mathrm{squash}(s_j)\)the class capsule's current output
4\(b_{ij}\;{+}{=}\;\hat u_{j|i}\cdot v_j\)agreement: votes pointing like \(v_j\) get boosted
Repeat steps 1–4 for \(r=3\) iterations. Then \(\|v_j\|\) is the class probability and \(c_{ij}\) is your Visualization-A heatmap.

Coupling \(c_{ij}\)

Bars = how much each lower capsule sends to each class. They start uniform (≈0.33) and commit as agreement builds.

The intuition: it's a small election that runs 3 times. Round by round, parts that vote alike reinforce each other, stray votes fade, and each part commits to the whole it best explains. Attention does this once; routing does it iteratively — the link to your Transformer comparison.

8 Counting the parameters

Two kinds of quantities — don't confuse them:

SymbolWhatLearned byCount (MNIST)
\(W_{ij}\)transform matrices (16×8 per part–class pair)backpropagation (fixed after training)8×16×1152×10 = 1,474,560
\(c_{ij}\)coupling coefficientsrouting at run-time (per input, NOT backprop)1152×10 = 11,520 (per image)

Why this matters for your project

\(W\) is the static wiring (like any weight matrix). \(c\) is the dynamic, input-dependent part — recomputed for every image. That dynamic \(c\) is exactly what makes this "routing," and it's the direct analogue of attention weights \(\alpha_{ij}\).

Conv1 adds \(256\times(9{\times}9{\times}1+1)=20{,}992\) params; Primary caps adds \(256\times(9{\times}9{\times}256+1)=5{,}308{,}672\).

9 Loss & classification

Margin loss (per class \(k\))

$$L_k=T_k\max(0,m^+-\|v_k\|)^2+\lambda(1-T_k)\max(0,\|v_k\|-m^-)^2$$ \(T_k=1\) if digit \(k\) is present. \(m^+=0.9,\ m^-=0.1,\ \lambda=0.5\).
Present class → length pushed above 0.9; absent → below 0.1.

Reconstruction (regularizer)

A small decoder rebuilds the image from the winning capsule. Adds a tiny MSE term that forces the capsule to encode real pose, not just a label.
Prediction: take the length of each digit capsule, \(\|v_j\|\). Longest wins. No softmax over logits — the geometry is the probability.
